# 行情刷新机制设计文档 (Market Data Refresh Strategy)

## 1. 概述
为了平衡数据的实时性、服务器负载以及接口反爬虫安全性，本系统采用了一套基于用户行为感知的**智能动态随机刷新机制**。

## 2. 核心设计原则
*   **去规律化**：消除固定的频率，模拟真实人类随机点击行为。
*   **环境感知**：根据用户是否正在观察页面，动态切换刷新强度。
*   **资源保护**：在非交易时间或页面不可见时，大幅降低请求频次。

## 3. 刷新逻辑详解

### 3.1 刷新频率策略
系统会根据 `document.hidden` 状态自动在两个不同的随机区间内切换：

| 页面状态 | 刷新区间 (Min - Max) | 设计意图 |
| :--- | :--- | :--- |
| **活跃 (Active)** | 3 分钟 - 6 分钟 | 保证用户在看盘时行情具备较好的实时性。 |
| **后台 (Hidden)** | 10 分钟 - 15 分钟 | 用户不在看盘。大幅降低请求频率，规避反爬风险。 |

### 3.2 随机偏移 (Jitter)
刷新时刻不再是“每隔 X 秒”，而是“在 [Min, Max] 区间内掷骰子”。这种完全随机的时间分布能有效绕过大部分基于时间序列规律检测的反爬虫算法。

### 3.3 链式递归延时
采用 `setTimeout` 递归而非 `setInterval`。
*   **setInterval 缺陷**：不顾请求耗时强制触发，易导致请求堆积或服务器压力。
*   **setTimeout 递归优点**：上一次请求完全结束（无论成功或失败）后，才开始计算下一次的等待时间，确保链路通畅且安全。

## 4. 交互增强 (Visibility API)

### 4.1 即时唤醒机制
系统监听浏览器的 `visibilitychange` 事件，其行为受代码中 `IS_WAKEUP_ENABLED` 常量控制：

*   **当 `IS_WAKEUP_ENABLED = true` 时（生产模式推荐）**：
    *   **即时触发**：当用户从其他标签页、最小化窗口或锁屏状态切回本页面时，系统会绕过定时器，立即执行一次行情刷新。
    *   **节奏重置**：同时会重置自动刷新定时器，将刷新频率立刻从后台长周期切换回“活跃模式”区间（3-6分钟）。
    *   **优点**：保证用户在回到页面的一瞬间看到的是最新行情，提供最佳的交互实时感。

*   **当 `IS_WAKEUP_ENABLED = false` 时（开发/静默模式）**：
    *   **静默切回**：用户切回页面时，系统不会发起即时请求，也不会重置现有计时器。
    *   **维持节奏**：系统将维持当前的随机刷新节奏（若处于后台状态则保持 10-15 分钟一次），直到下一次自然随机时刻到来。
    *   **优点**：极大地减少了在频繁进行代码调试、查阅文档等切换操作时的无效接口调用，避免被反爬系统识别，降低开发环境的干扰。

## 5. 鲁棒性与安全

### 5.1 非交易时段熔断
自动刷新逻辑会检查 `isTradingTime()`。
*   **熔断触发**：在 A 股非交易时段（如 15:05 之后或周末），自动刷新会静默跳过请求，不产生系统开销。

### 5.2 内存管理
在 Vue 3 生命周期中严格管理监听器与定时器：
*   `onMounted`: 注册 `visibilitychange` 监听，启动定时器。
*   `onBeforeUnmount`: 手动移除 `addEventListener` 挂载的事件，清理 `setTimeout` 指针。
*   **效果**：确保组件销毁后，没有任何后台脚本残留或内存泄漏。
