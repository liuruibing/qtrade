# `__update_bi` 下，什么时候“一笔”会锁定、不再延伸？

本文从 `CZSC.__update_bi(self)` 的实际代码行为出发，回答你关心的“笔什么时候会锁定（端点不再被修改/撤销）、什么时候还会延伸（回滚重算）”。

相关源码位置（当前版本）：

- `CZSC.__update_bi`: `czsc/py/analyze.py:220-263`
- “破坏/回滚”逻辑：`czsc/py/analyze.py:254-262`

---

## 1) 这份实现里没有显式的“锁定”标记

`BI` 对象里没有 `locked=True/False` 之类字段。所谓“锁定/不再延伸”，在这份实现中是一个**隐含状态**：

- **只有 `bi_list` 的最后一笔**（`last_bi = self.bi_list[-1]`）会被拿来做“破坏检查”，从而可能被撤销并回滚重算（`czsc/py/analyze.py:254-262`）。
- 任何**不是最后一笔**的历史笔，在 `__update_bi` 里都不会被检查、不会被 pop、也不会被修改其端点。

所以，“锁定”可以等价理解为：

> 在当前时刻，一笔如果不是 `self.bi_list[-1]`，那么在这套算法里它就是“锁定的”（不会再延伸/回滚）。

---

## 2) “还会延伸”的笔：始终只有最后一笔

`__update_bi` 的后处理逻辑是：

```python
last_bi = self.bi_list[-1]
bars_ubi = self.bars_ubi
if (last_bi.direction == Up and bars_ubi[-1].high > last_bi.high) \
        or (last_bi.direction == Down and bars_ubi[-1].low < last_bi.low):
    self.bars_ubi = last_bi.bars[:-2] + [x for x in bars_ubi if x.dt >= last_bi.bars[-2].dt]
    self.bi_list.pop(-1)
```

它表达的语义是：

- **只要后续价格突破了最后一笔的极值**，就认为“最后一笔终点不成立”，撤销这笔并把K线回并回去，等待下一次 `check_bi` 重新生成（通常会生成一根更长/端点更靠后的同向笔）。

因此，“会延伸”的范围永远在序列右端：**尾部一笔是可变的**。

---

## 3) “什么时候锁定”：当它不再是最后一笔时

一笔从“可延伸”变为“锁定”，发生在它的右侧出现并保留下来一根后续笔时，即：

- `check_bi(bars_ubi)` 成功返回了一个新 `BI`，并被 append 到 `self.bi_list`（`czsc/py/analyze.py:249-252`）
- 此时，原来的最后一笔变成了倒数第二笔
- 回滚逻辑只会检查新的 `self.bi_list[-1]`，不会再触碰倒数第二笔

用一句话概括：

> 在这份实现里，“锁定”发生在**下一笔被确认并追加之后**：上一笔成为历史笔，从此不再被延伸/回滚（除非它再次变回最后一笔，见下一节）。

---

## 4) 重要细节：锁定并非“永久”，它可能被“解锁”（重新变成最后一笔）

虽然历史笔不会被直接修改，但存在一种情况会让“已锁定的笔”重新回到尾部，从而再次可被延伸：

1) 假设已有 `... , bi_{n-1}, bi_n`  
2) 如果 `bi_n` 发生“破坏”，会被 `pop(-1)` 撤销（`czsc/py/analyze.py:257-262`）
3) 那么 `bi_{n-1}` 会重新变成 `self.bi_list[-1]`
4) 之后只要新的行情继续突破 `bi_{n-1}` 的极值，它也可能再被撤销并重算（因为它又成了“最后一笔”）

所以更精确的表述是：

- **时刻 t：**非最后一笔 = 锁定（不会动）
- **时刻 t+Δ：**如果后面的笔被连续撤销，原先的历史笔可能再次成为最后一笔，于是“解锁”，重新进入可延伸状态

这也是很多在线缠论实现的共同特征：**只对尾部结构允许回溯**。

---

## 5) 针对你画图时的预期

从 `__update_bi` 的机制推导，图上应当满足：

- 在持续喂数据的过程中：**最后一笔（以及可能的最后两三笔）可能会变**（端点漂移、撤销再生成）
- 越靠左的历史笔：越稳定（因为要“解锁”它，必须先把它右侧的所有笔都撤销掉）
- 如果你跑的是一段离线历史数据：最终结果里通常只有**最右侧最后一笔**仍可被认为“未完全锁定”（因为再来新K线仍可能触发破坏回滚）
